# M8 更多驱动：AWS S3 与百度网盘接入设计

> 状态：已实现并测试通过（见 §8）。
> 分支：`feat(driver)/m8-s3-baidu`（从 main M7 合并后基线切出）。

## 1. 解决的问题

M0–M7 只有 `Local` 一个驱动，存储只能是宿主机本地目录。M8 的本轮目标是落地两个**远程驱动**，验证远程驱动接入范式并扩大可挂载存储类型：

1. **AWS S3（及 S3 兼容，如 MinIO/阿里云 OSS 兼容端点）**：读 + 写（list/get/link/mkdir/put/remove/rename/move/copy）。
2. **百度网盘（Baidu Netdisk）**：读 + 下载（list/get/link）。

不在本轮范围（按 `development-plan.md` 的 M8 优先级裁剪，避免一次摊太大）：WebDAV/S3 **服务端协议**、阿里云盘/OneDrive 驱动、百度网盘**写**（切片上传，见 §7）。

## 2. 为什么这样选（关键约束）

- **驱动 SPI 已就绪，零侵入扩展**：新增驱动只需在 `infrastructure/driver/<name>/` 实现 `StorageDriverFactory` + 一个实现 `StorageDriver`(=`DriverMeta`+`DriverReader`) 的类，`StorageDriverRegistry` 自动按 Spring Bean 收集；管理端表单已能按 `DriverInfo.items` **动态渲染**配置项（见 `web/src/pages/admin/Storages.tsx`）。因此**后端加驱动即前端可配，无需改前端**。
- **`link()` 的两种取链策略**：
  - **S3 → 预签名重定向**：`link()` 返回 `presignGetObject` 短期 URL，`DownloadController` 既有逻辑对非 `file` 协议直接 302，零额外带宽。
  - **百度 → 鉴权代理**：百度 `dlink` 下载要求特定 `User-Agent`（`pan.baidu.com`），浏览器 302 会丢该头而失败，必须**由服务端带头流式代理**。为此给 `DownloadController` 增加"当 `FileLink.headers()` 非空则代理流式拉取"的分支（既有 `FileLink` 已含 `headers` 字段，复用之）。
- **依赖最小化**：
  - **S3**：引入 **AWS SDK v2（`software.amazon.awssdk:s3`，经 BOM 管理版本）**——这是**唯一新增依赖**，需用户批准（已确认）。它把 SigV4 签名、分页、预签名、错误模型都给齐，自己手搓 SigV4 风险高、收益低。默认同步客户端走 apache-client，不引 netty。
  - **百度**：**不引新依赖**，用 JDK 内置 `java.net.http.HttpClient` + 既有 Jackson 调 REST/JSON。
- **凭据存储复用 `storages.addition`**：各驱动私有配置（密钥/桶/refreshToken 等）以 JSON 存既有 `addition` 字段，`init()` 时反序列化，**无新表、无新迁移**。

## 3. 总体架构

```
FsApplicationService（不变）
  └─ MountedStorageRegistry.createRuntime → factory.create() → driver.init(ctx)
        ├─ LocalDriver          (既有)
        ├─ S3Driver             (新) AWS SDK v2 S3Client + S3Presigner
        └─ BaiduDriver          (新) JDK HttpClient + token 缓存

下载取链:
  DownloadController /d 与 /sd
    └─ FsApplicationService.link → driver.link(file)
         ├─ S3   : FileLink(url=presigned https, headers={})        → 302 重定向
         └─ Baidu: FileLink(url=dlink+token, headers={UA:pan.baidu.com}) → 代理流式(新分支)
```

新增/变更模块：

```
pom.xml                          (改) dependencyManagement 引入 awssdk BOM + 依赖 s3
infrastructure/driver/s3/
  S3DriverFactory.java           (新) name()="S3"，info() 声明配置项
  S3Driver.java                  (新) StorageDriver+DriverGetter+DriverRootProvider+DriverWriter
  S3DriverAddition.java          (新) bucket/endpoint/region/ak/sk/rootFolder/pathStyle/signExpireSec
infrastructure/driver/baidu/
  BaiduDriverFactory.java        (新) name()="BaiduNetdisk"
  BaiduDriver.java               (新) StorageDriver+DriverGetter+DriverRootProvider（只读）
  BaiduDriverAddition.java       (新) refreshToken/clientId/clientSecret/rootPath
  BaiduTokenManager.java         (新) refresh_token → access_token，内存缓存 + 过期刷新
api/controller/
  DownloadController.java        (改) link.headers 非空 → 代理流式拉取（转发 Range）
common/exception/
  ErrorCode                      (改) 新增 DRIVER_REMOTE_ERROR（远程驱动 I/O/鉴权失败映射）
```

## 4. 实现方式（分提交批次）

**批次 1 `chore(db)`/`feat(storage)`：依赖 + S3 驱动**
- `pom.xml`：`dependencyManagement` import `software.amazon.awssdk:bom:2.28.x`，`dependencies` 加 `s3`。
- `S3DriverAddition` + `S3DriverFactory.info()`（配置项见 §5）。
- `S3Driver`：
  - `init()`：构建 `S3Client`/`S3Presigner`（`region`、可选 `endpointOverride`、`forcePathStyle`、静态凭据），`headBucket` 验证连通；失败抛 `BusinessException` → 存储置 `init_error`。
  - 读：`list`（`ListObjectsV2` + `delimiter="/"`，`CommonPrefixes`→目录、`Contents`→文件，处理 continuation 分页）、`get`（`headObject`；404 再探目录前缀）、`getRoot`、`link`（`presignGetObject`）。
  - 写：`mkdir`（写 `key+"/"` 空对象）、`put`（`PutObject`，size 已知用 `fromInputStream(in,len)`）、`remove`（文件 `deleteObject`；目录列出前缀批量 `deleteObjects`）、`rename/move/copy`（`CopyObject` 单文件 / 目录递归 copy[+del]）。
- 单测：key↔actualPath 映射、配置解析、错误映射（mock 或纯逻辑）。

**批次 2 `feat(storage)`：百度网盘驱动 + 代理下载**
- `BaiduDriverAddition` + `BaiduDriverFactory.info()`。
- `BaiduTokenManager`：`refresh_token` 换 `access_token`（`openapi.baidu.com/oauth/2.0/token`），按 `expires_in` 内存缓存并提前刷新；百度 refresh_token 长期有效、可重复使用，**无需回写**。
- `BaiduDriver`（只读）：
  - `init()`：解析 addition，预取一次 token 验证可用。
  - `list`：`xpan/file?method=list&dir=...` → 映射 `server_filename/isdir/size/server_mtime`，`fs_id` 存入 `FileObject.id()`。
  - `get`：列父目录匹配名（百度无按路径 stat）；root 直接返回目录对象。
  - `link`：`xpan/multimedia?method=filemetas&fsids=[fsid]&dlink=1` 取 `dlink` → `FileLink(url=dlink+"&access_token=", headers={User-Agent: pan.baidu.com})`。
- `DownloadController`：`link.headers()` 非空 → 用 JDK `HttpClient` 带头拉取并流式回写，转发客户端 `Range`、回传上游 `Content-Range/Length`、跟随 302。
- 单测：token 刷新逻辑、list JSON 解析、link 头注入；代理分支用本地 stub server 验证头透传与 Range。

**批次 3 `docs`：文档**
- 更新 README 驱动列表、`development-plan.md`（M8 部分落地 S3+Baidu），回填本文 §8，补 `.env.example`/配置说明（凭据走 `addition`，不入全局配置）。

## 5. 关键配置与数据流

### 驱动配置项（`DriverInfo.items` → 管理端动态表单 → 存 `addition` JSON）

**S3（`driver="S3"`）**

| name | type | 必填 | 说明 |
| --- | --- | --- | --- |
| `bucket` | string | 是 | 桶名 |
| `endpoint` | string | 否 | S3 兼容端点；空=AWS 官方 |
| `region` | string | 是 | 区域，默认 `us-east-1` |
| `accessKeyId` | string | 是 | Access Key |
| `secretAccessKey` | string | 是 | Secret Key |
| `rootFolder` | string | 否 | 桶内根前缀，默认空（桶根）|
| `pathStyle` | boolean | 否 | 路径风格寻址（MinIO 常需）默认 false |
| `signExpireSec` | integer | 否 | 预签名有效期秒，默认 900 |

**百度网盘（`driver="BaiduNetdisk"`）**

| name | type | 必填 | 说明 |
| --- | --- | --- | --- |
| `refreshToken` | string | 是 | OAuth refresh_token（长期有效）|
| `clientId` | string | 是 | 应用 AppKey |
| `clientSecret` | string | 是 | 应用 SecretKey |
| `rootPath` | string | 否 | 网盘内根路径，默认 `/`（个人应用通常 `/apps/<应用名>`）|

> 安全：以上密钥仅存 `addition`，不写入全局配置/日志；当前前端 secret 字段以普通文本框渲染（限制见 §7）。

### S3 key 映射

```
actualPath "/"        → prefix = norm(rootFolder)            （列桶根/前缀）
actualPath "/a/b.txt" → key    = join(rootFolder, "a/b.txt")
目录             ↔ key 以 "/" 结尾（mkdir 写空对象占位）
list: ListObjectsV2(prefix, delimiter="/") → CommonPrefixes=目录, Contents=文件(去占位/去前缀自身)
link: presignGetObject(bucket,key,signExpireSec) → https URL → DownloadController 302
```

### 百度下载数据流（代理）

```
GET /d/<path>?sign=...  （登录态；分享下载同理走 /sd）
  1. FsApplicationService.link → BaiduDriver.link
       filemetas(fsid, dlink=1) → dlink
       return FileLink(url = dlink + "&access_token=" + token,
                       headers = { "User-Agent": "pan.baidu.com" })
  2. DownloadController: headers 非空 → HttpClient.send(GET url, 注入 headers + 透传 Range)
       回写 上游 status(200/206) + Content-Range/Length/Type + body
```

## 6. 异常处理

| 场景 | 错误码 |
| --- | --- |
| 驱动配置缺失/非法（addition 解析失败、必填空）| `BAD_REQUEST` |
| 桶/凭据无效（headBucket 失败、token 刷新失败）| 抛 `BusinessException` → 存储置 `init_error` |
| 远程列目录/取链 I/O 或鉴权失败 | `DRIVER_REMOTE_ERROR`（新，502/500）|
| 对象不存在 | `OBJECT_NOT_FOUND` |
| 百度驱动调用写接口（不支持）| `DRIVER_NOT_SUPPORTED`（未实现 `DriverWriter`，应用层既有映射）|
| 代理下载上游非 2xx | 透传/映射为 `DRIVER_REMOTE_ERROR` |

## 7. 风险、限制、后续

- **新依赖（重要）**：引入 AWS SDK v2 `s3`（含 apache-client 等传递依赖，体积约数十 MB）。已确认批准。S3 兼容服务（MinIO/OSS）通过 `endpoint`+`pathStyle` 适配，但各家兼容度不一，错误码映射尽力而为。
- **百度只读（重要）**：本轮**不实现百度写**。其上传为 `precreate → superfile2 分片 → create` 多步流程，分片大小敏感、无真实账号难以端到端测试，价值/复杂度不成正比。`BaiduDriver` 不实现 `DriverWriter`，应用层自动映射为 `DRIVER_NOT_SUPPORTED`；待有联调账号再补（框架已留位）。
- **百度代理带宽**：百度下载必须服务端代理，**占用主服务出入带宽**，且受百度侧限速/鉴权波动影响；大文件下载稳定性弱于 S3 重定向。
- **token 生命周期**：百度 access_token 内存缓存，多实例各自持有、各自刷新（refresh_token 可重复使用，无冲突）；进程重启即重新换取。Aliyun 式会轮换 refresh_token 的驱动若后续接入，需回写 `addition`（本轮不涉及）。
- **S3 无真实目录**：以 `/` 前缀模拟目录；写操作不做"目标已存在"检查（S3 覆盖语义），与 Local 的拒绝覆盖行为不同。
- **前端 secret 明文**：配置表单当前对 string 类型渲染普通输入框，密钥明文可见；后续可加 `password`/secret 类型渲染（独立小改）。
- **大文件上传**：S3 `put` 当前按已知 size 单次上传；超大文件 multipart 与 M6 任务中心的异步上传通道整合留作后续。

## 8. 测试结果（已实现）

S3/百度均依赖外部服务，CI 无真实凭据，故以**纯逻辑单测 + 本地 stub server**为主；连通性（headBucket/真实 list/302 重定向/真实代理下载）作为本地手测验收项（见 PR）。

| 测试类 | 覆盖点 |
| --- | --- |
| `S3DriverTest` | key↔actualPath 映射（空/非空 rootFolder，往返一致）、addition 缺省回退、空 addition 拒绝 |
| `BaiduTokenManagerTest` | token 有效期内缓存命中（仅 1 次上游请求）、扣除 skew 后过期则每次刷新 |
| `BaiduDriverTest` | list JSON 解析（文件/目录、`fs_id`→id、`path`→actualPath）、link 注入 `User-Agent: pan.baidu.com` 且 dlink 带 access_token、rootPath 路径互转 |
| `ProxyDownloadTest` | 代理分支：UA 头透传上游、Range 透传并回写 206 + `Content-Range`、无 Range 回写 200 全量 |

实现与设计的差异/补充：
- 新增驱动后 `S3DriverFactory`/`BaiduDriverFactory` 被 `StorageDriverRegistry` 自动收集，`/api/admin/driver/list` 现返回三个驱动（按名排序 `BaiduNetdisk`/`Local`/`S3`）；相应更新了 `AdminStorageControllerTest` 的断言为顺序无关。
- 代理下载分支落在 `DownloadController.serveTarget`：本地 `file` 协议走分段流，远程且 `link.headers()` 非空走带头代理流式拉取，其余远程 302（S3 预签名走此路径）。

结果：`mvn test` 全绿（74 个测试，0 失败/0 错误）；`npm --prefix web run build` 通过（管理端按 `DriverInfo.items` 动态渲染 S3/百度配置表单，无需改前端）。

验收标准达成：管理端可配置并启用 S3 / 百度网盘存储（连通失败置 `init_error`）；S3 经预签名 302 下载、可上传/删除；百度可 list 并经服务端代理（带 UA）下载；纯逻辑与 stub 覆盖的行为全绿。真实账号连通性为本地手测项。
