# M7 分享与公开访问设计

> 状态：设计稿（待批准后实现）。
> 分支：`feat(share)/m7-share`（从 main 当前基线切出）。

## 1. 解决的问题

M0–M6 完成后系统是一个登录态网盘：任何下载/浏览都要先登录并经 basePath + 权限校验。M7 引入**免登录的公开分享**：

1. 登录用户可把自己可见的某个文件/目录创建为**公开分享链接**。
2. 链接支持**密码、过期时间、访问次数上限、阅后即焚、是否允许预览、是否允许下载**等控制。
3. 匿名访客凭 `shareId`（+ 密码）即可浏览分享目录、查看文件详情、下载文件，全程**不需要登录账号**。

不在本阶段范围：更多驱动/协议（M8）、AI 语义检索（M9）。在线预览渲染交由前端用 `allowPreview` 开关 + 既有下载流实现，后端不做格式转换。

## 2. 为什么这样选（关键约束）

- **复用既有表与下载链路**：`shares` 表 + `ShareEntity`/`ShareMapper` 已在 `V1` 建好（见 `V1__init_schema.sql`），本阶段**不需要新迁移**，仅填充与查询。公开下载复用 `FsApplicationService.link` 与 `DownloadController` 的本地/重定向输出逻辑。
- **不引入新依赖**：分享令牌复用既有 **HMAC-SHA256**（与 `DownloadSignService` 同款，密钥取 jwt secret），不引入新的会话/缓存组件；密码哈希复用既有 `PasswordService`（BCrypt）。
- **以"创建者身份 + 根路径约束"做权限**：公开访问没有 `CurrentUser`。M7 在访问时按分享记录**重建一个合成的创建者上下文**（创建者 basePath/permission/roles），并把访客可达路径**夹在 `share.rootPath` 子树内**，从而最大化复用既有 `FsApplicationService` 的存储解析与读逻辑，不另写一套文件访问代码。
- **沿用分层**：`ShareController`(/api/share/**) / `PublicShareController`(/api/public/share/**) / 下载 `/sd/{shareId}/**` → `ShareApplicationService` → `ShareMapper` + 复用 `FsApplicationService`，与 storage/meta/task 模块同构。

### 权限与安全边界（重要）

- 分享访问**不再校验目录 Meta 密码**（分享者已显式决定公开该子树），但**仍套用 Meta 的 hide 隐藏规则**，避免分享目录里泄漏本应隐藏的条目。
- 访客路径一律 `PathUtils.sanitizePath` 归一并**强制前缀校验**：任何越出 `share.rootPath` 的路径直接 `PERMISSION_DENIED`，防止借分享做路径穿越读到创建者其他文件。
- 合成创建者上下文**只读**：公开接口只暴露 list/get/link，绝不暴露写操作。

## 3. 总体架构

```
管理（登录态）:
  ShareController(/api/share/**) ─► ShareApplicationService ─► ShareMapper(shares)
     create/update/delete/list           （创建时校验 rootPath 对当前用户可读）

公开（匿名）:
  PublicShareController(/api/public/share/**)
     info ─► 返回分享元信息（是否需要密码/是否允许下载/过期等，不含敏感字段）
     auth ─► 校验密码 → 签发 shareToken（HMAC，绑定 shareId+exp）
     list ─► 校验 shareToken + 访问控制 → 合成创建者上下文 → FsApplicationService.list（夹在 rootPath 内）
     get  ─► 同上 → FsApplicationService.get
                              │
  下载 /sd/{shareId}/**  ─────┘ → 校验 shareToken + allowDownload → FsApplicationService.link
                                   → 复用 DownloadController 的本地流/重定向输出
```

## 4. 实现方式（分提交批次）

**批次 1 `feat(share)`：分享管理（登录态 CRUD）**
- `Share` 领域模型 + `ShareModelMapper`；`ShareApplicationService.create/update/delete/list`。
- 创建时：用当前用户校验 `rootPath` 可读（复用 `FsApplicationService.get` / 权限解析），生成 URL 安全随机 `shareId`（≥ 22 字符 base64url），密码经 `PasswordService` 哈希存 `password_hash`（`password_salt` 留作兼容字段）。
- 所有权校验：update/delete/list 仅作用于 `creator_id == 当前用户`。
- `ShareController` + 请求/响应 DTO。单测：创建、列表、所有权 403、不可读 rootPath 拒绝。

**批次 2 `feat(share)`：公开访问 + 访问控制**
- `ShareTokenService`（HMAC 绑定 `shareId:exp`，TTL 可配 `asuka.share.token-ttl-seconds`）。
- `ShareAccessService`：`enabled`、`expires_at`、`access_limit/access_count`、`burn_after_read` 统一校验与原子计数。
- `PublicShareController.info/auth/list/get`：
  - `info`：返回 `name/isDir/needPassword/allowPreview/allowDownload/expired` 等**非敏感**字段。
  - `auth`：校验密码 → 计数/控制 → 签发 `shareToken`。
  - `list/get`：校验 `shareToken` → `ShareAccessService` 控制 → 合成创建者上下文 → 复用 `FsApplicationService`，路径夹在 `rootPath` 内 + 套用 hide。
- `SecurityWebMvcConfig` 放行 `/api/public/share/**`。单测：密码分享、过期分享、阅后即焚、次数上限。

**批次 3 `feat(share)`：分享下载 `/sd/{shareId}/**`**
- 在 `DownloadController`（或新 `ShareDownloadController`）加 `/sd/{shareId}/**`：校验 `shareToken` + `allowDownload` → `ShareApplicationService.link`（合成上下文）→ 复用既有本地流/重定向 + Range 输出。
- `SecurityWebMvcConfig` 放行 `/sd/**`。单测：禁止下载、token 缺失/失效、Range 分段。

**批次 4 `feat(web)`/`docs`：前端 + 文档**
- 前端：分享管理页（创建/更新/删除/列表，设密码/过期/次数/预览/下载开关）；公开分享页（密码校验入口 → 目录浏览 → 下载/预览）。
- 更新 README、`development-plan.md`（M7 ✅），回填本文 §8。

## 5. 关键接口与数据流

### 接口签名（HTTP）

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| `POST` | `/api/share/create` | 登录（rootPath 可读）| 创建分享，返回 shareId |
| `POST` | `/api/share/update` | 所有者 | 更新分享控制项 |
| `POST` | `/api/share/delete` | 所有者 | 删除分享 |
| `GET`  | `/api/share/list` | 登录 | 我的分享（分页）|
| `GET`  | `/api/public/share/info` | 匿名 | 分享元信息（是否需密码等）|
| `POST` | `/api/public/share/auth` | 匿名 | 密码校验 → 签发 shareToken |
| `POST` | `/api/public/share/list` | 匿名 + shareToken | 分享目录列表（分页）|
| `POST` | `/api/public/share/get`  | 匿名 + shareToken | 分享文件详情 |
| `GET`  | `/sd/{shareId}/**` | 匿名 + shareToken | 分享下载（Range）|

### 访问控制判定（每次公开访问）

```
resolve(shareId):
  1. 查 shares；不存在 → SHARE_NOT_FOUND
  2. enabled=false           → SHARE_NOT_FOUND（对外不区分禁用/不存在）
  3. expires_at 已过         → SHARE_EXPIRED
  4. access_limit>0 且 access_count>=limit → SHARE_EXPIRED
auth(shareId, password):
  5. 有密码且不匹配 → SHARE_PASSWORD_INCORRECT；缺密码 → SHARE_PASSWORD_REQUIRED
  6. 通过 → 计数 access_count+1（原子 UPDATE ... WHERE）
  7. burn_after_read=true → 本次之后置 enabled=false（首次访问即失效）
  8. 签发 shareToken = HMAC(shareId:exp)
list/get/download:
  9. 校验 shareToken（绑定 shareId 且未过期）；失败 → SHARE_TOKEN_INVALID
 10. download 还需 allow_download=true，否则 SHARE_DOWNLOAD_DISABLED
```

> 计数口径：以 `auth` 成功（"打开分享"）计一次访问；list/get/download 凭已签发 token 不再计数，避免目录浏览把次数瞬间耗尽。阅后即焚在首次 `auth` 后失效。

### 公开路径解析数据流

```
访客访问 /api/public/share/list { shareId, subPath, password? } (带 shareToken)
  1. resolve(shareId) + 校验 shareToken
  2. visiblePath = join(share.rootPath, sanitize(subPath))；越界 rootPath → PERMISSION_DENIED
  3. ctx = 合成 CurrentUser(创建者 basePath/permission/roles)
  4. FsApplicationService.list(ctx, visiblePath) → 套用 Meta hide → 去敏返回
  5. 下发的每个文件项的下载地址 = /sd/{shareId}/{相对 rootPath 的子路径}
```

## 6. 异常处理

| 场景 | 错误码 |
| --- | --- |
| 分享不存在 / 已禁用 | `SHARE_NOT_FOUND`（404）|
| 分享过期 / 超访问次数 | `SHARE_EXPIRED`（410）|
| 需要密码但未提供 | `SHARE_PASSWORD_REQUIRED`（401）|
| 密码错误 | `SHARE_PASSWORD_INCORRECT`（401）|
| shareToken 缺失/失效 | `SHARE_TOKEN_INVALID`（401）|
| 分享禁止下载仍请求 `/sd` | `SHARE_DOWNLOAD_DISABLED`（403）|
| 访问/修改他人分享 | `PERMISSION_DENIED` |
| 创建分享的 rootPath 当前用户不可读 | `PERMISSION_DENIED` |
| 子路径越出 rootPath | `PERMISSION_DENIED` |

## 7. 风险、限制、后续

- **限制**：访问计数与阅后即焚为**单库原子 UPDATE**，单实例强一致；多实例下并发 `auth` 仍由 `UPDATE ... WHERE access_count<limit` 行级条件兜底，不引入分布式锁。
- **限制**：合成创建者上下文取创建分享时刻的创建者权限；若分享存续期间创建者被降权/删号，访问可能仍按旧上下文放行——M7 在 `resolve` 时**实时重载创建者**，创建者被禁用则分享失效。
- **限制**：在线预览由前端基于 `allowPreview` + 下载流实现，后端不转码；大文件预览即走下载链路。
- **风险**：分享 token 复用 jwt secret 的 HMAC，secret 轮换会使存量 shareToken 立即失效（可接受，访客重新输密码即可）。
- **后续**：分享访问审计日志、目录密码与分享密码分离策略、短链/二维码、限速防爬。
